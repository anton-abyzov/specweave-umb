# Glossary - Index by Category

Browse SpecWeave terminology organized by topic.

## Core Concepts

| Term | Description |
|------|-------------|
| **Acceptance Criteria (AC)** | Specific, measurable conditions that must be met for a user story to be complete |
| **Increment** | A unit of work representing a feature, enhancement, or fix with unique ID |
| **Living Docs** | Self-updating documentation synchronized with code |
| **SpecWeave** | Spec-driven Skill Fabric for AI coding agents with autonomous workflows |
| **User Story (US)** | High-level feature description from user's perspective |

## Development & Architecture

| Term | Description |
|------|-------------|
| **ADR (Architecture Decision Record)** | Document capturing important architectural decisions |
| **Hook** | Script executing automatically at specific workflow points |
| **Task** | Specific, actionable work item within an increment |
| **TDD (Test-Driven Development)** | Development approach where tests are written first |
| **Model Hints** | Task annotations for AI model selection |
