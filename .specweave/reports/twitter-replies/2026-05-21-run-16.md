**Why this answer:**
- **Concede the strongest part.** "fair." — same pattern as runs 12-15. Four hops in a row, "concede then split" has become the locked move. Picks up Matt's exact framing: `cluster 3 day-1 ≠ cluster 3 week-4`, `math hides the meaning-shift`, `before the math finishes type-checking`. Reusing the opponent's language is the highest-trust conversation signal.
- **Version the embedding.** The conceptual move. The previous solution (embed, don't label) treated the embedding as a static structure. Make the embedding itself a versioned artifact — same treatment as skills, manifests, canaries. Once you accept embedding versioning, drift becomes a first-class observable.

Concrete, implementable, and answers "how do you know it drifted?" — by measuring cluster centroid movement against the day-1 baseline. This is a standard ML-ops pattern (embedding drift detection) but it has not been brought into the agent-skill-eval discussion explicitly until now.
- **interval = f(drift).** The math response to Matt's "interval widens silently" critique. Make widening a function of measurable drift instead of a hidden state. The interval is no longer silent — it's a published function of an audited variable.
- **drift-alarm fires before the math finishes type-checking.** The closer that directly inverts Matt's failure mode. Where Matt said "nothing notices because math type-checks," this says the alarm runs in parallel and fires faster than the type-check completes. Aphorism candidate.
