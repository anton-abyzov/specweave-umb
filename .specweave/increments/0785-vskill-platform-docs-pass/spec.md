# Feature: vskill-platform docs pass — readability, counts hardening, copy refresh

## Overview

The gold "Certified" trust-tier card on `/trust` and `/docs/getting-started` renders body text in `#666` on a `#FEF9C3` background — measured ~3.5:1, fails WCAG AA (needs 4.5:1 for body). Same pairing repeats in `TierBadge.tsx`, `SearchPalette.tsx`, and other gold-bg surfaces. Light theme only — dark theme uses transparent gold and is fine.

While auditing, three adjacent quality issues surfaced and are bundled into this single docs pass:

1. **Hardcoded counts bypass the auto-generator.** `src/lib/generated-counts.ts` is regenerated by `scripts/sync-agents-json.cjs` on `prebuild`, but multiple pages still embed literals (`"52 static patterns"`, `"53 platforms"`, `"8 domain plugins"`, `"14 expert skills"`). vskill ships **53** patterns today; COUNTS file is one regen behind, and the prose copies are completely disconnected from source-of-truth.
2. **Stale time-bound copy.** `/insights/claude-code/page.tsx:392` references "April 1-7 2026" and "May 2026" launch windows that are now past or current — they read as overdue. `/docs/faq/page.tsx:102` claims a "24-hour" report-review SLA that has never been operationally validated.
3. **Tier vocabulary drift.** Internal `T0–T4` enum, public 3-tier ladder ("Scanned/Verified/Certified"), and parallel semantic naming ("Security-Scanned/Trusted Publisher/Tainted") all coexist across `TrustBadge.tsx`, `TierBadge.tsx`, and FAQ copy — the 5-tier internal model leaks into UX inconsistently.

**Outcome:** All trust-tier surfaces readable in both themes; every numeric claim sourced from `COUNTS.*`; time-bound copy refreshed; single user-facing tier vocabulary across all 11 public pages. No doc-framework migration — the current Next.js `app/docs/*` structure stays.

## Out of scope

- Doc framework migration (Docusaurus / Mintlify / Nextra) — deferred
- Active increment 0784 (studio Install button) — leave its dirty `spec.md` alone
- Trust-tier scoring logic / scanner behavior — copy and presentation only
- Backwards renaming of `T0–T4` enum — internal model stays; only user-facing labels harmonize

## User Stories

### US-001: Readable Certified trust-tier cards in light theme
**Project**: vskill-platform

**As a** vskill skill consumer reading `/trust` or `/docs/getting-started` in light theme
**I want** the gold "Certified" card body text and "HIGHEST" rank label to be clearly legible
**So that** I can understand the highest-trust tier without straining or zooming in

**Acceptance Criteria**:
- [x] **AC-US1-01**: `.tier-card[data-tier="certified"] .tier-card-desc` body text contrast ratio against `--tier-certified-bg` is ≥ 4.5:1 in light theme (WCAG AA body).
- [x] **AC-US1-02**: `.tier-card[data-tier="certified"] .tier-card-rank` ("HIGHEST") contrast ratio is ≥ 3:1 in light theme (WCAG AA large text).
- [x] **AC-US1-03**: Dark theme rendering of the same selector is unchanged or equally accessible — `preview_screenshot` of dark-mode `/trust` matches the pre-change baseline visually.
- [x] **AC-US1-04**: The same readability fix propagates to every gold-bg consumer: `.tier-pill[data-tier="certified"]`, `TierBadge.tsx` (CERTIFIED variant), and `SearchPalette.tsx` (any gold tier indicator).
- [x] **AC-US1-05**: Verified-tier card receives parallel token discipline (`--tier-verified-text`) so future palette shifts don't regress contrast there either.

### US-002: All numeric claims driven by `generated-counts.ts`
**Project**: vskill-platform

**As a** docs maintainer
**I want** every prose mention of "X patterns / Y plugins / Z agents / N skills" to import from `COUNTS`
**So that** running `npm run prebuild` is the single point of truth and copy can never drift from reality

**Acceptance Criteria**:
- [x] **AC-US2-01**: After `npm run prebuild`, `src/lib/generated-counts.ts` reports `scanPatterns: 52` (current truth in vskill source — verified by generator's strict `^\s*id:\s*"` regex on `vskill/src/scanner/patterns.ts`).
- [x] **AC-US2-02**: All numeric literals in copy across these files are replaced with `${COUNTS.X}` interpolations: `src/app/layout.tsx`, `src/app/docs/page.tsx`, `src/app/docs/faq/page.tsx`, `src/app/docs/getting-started/page.tsx`, `src/app/docs/submitting/page.tsx`, `src/app/docs/plugins/page.tsx`, `src/app/docs/security-guidelines/layout.tsx`, `src/app/docs/security-guidelines/page.mdx` (body content, not YAML frontmatter), `src/app/components/AnimatedTerminal.tsx`, `src/app/components/home/PipelineFlow.tsx`, `src/app/components/homepage/FeatureSecurityVerified.tsx`, `src/app/insights/claude-code/page.tsx`, `src/app/skills/[owner]/[repo]/[skill]/security/page.tsx`. - [x] **AC-US2-03**: After `npm run build`, `grep -E '\b(52|53|14|8)\s+(static|patterns|platforms|plugins|skills|agents)\b' .next/server/app/**/*.html` returns zero hits — every count rendered to HTML traces back to `COUNTS`.
- [x] **AC-US2-04**: `npm run typecheck` passes — all new `import { COUNTS } from "@/lib/generated-counts"` statements resolve.

## Functional Requirements

### FR-001: Tier-scoped CSS color tokens
Add `--tier-certified-text` and `--tier-certified-rank` to the light-theme `:root` block in `globals.css`. Apply only inside `.tier-card[data-tier="certified"]` and `.tier-pill[data-tier="certified"]` selectors — the global `--text-muted` / `--text-faint` palette is untouched. Mirror with `--tier-verified-text` for token-discipline parity.

### FR-002: COUNTS import discipline
Every doc page that renders a count from `{52, 53, 14, 8, 9}` adjacent to `{patterns, plugins, skills, agents, platforms, dimensions, categories}` MUST import `COUNTS` and interpolate the value. Markdown emphasis is preserved around the dynamic value (e.g. `**${COUNTS.scanPatterns} static patterns**`).

### FR-003: Past-tense for elapsed launch windows
Time-bound copy with windows in the past (relative to today, 2026-04-27) is rewritten in past tense or removed. Future windows stay future-tense.

### FR-004: TierBadge label parity
`TierBadge.tsx` semantic-label map updates to use public ladder labels; the tier's icon and color stay unchanged.

## Non-Functional Requirements

- **Accessibility**: WCAG 2.1 AA contrast for body text inside all tier surfaces (≥4.5:1) and large/label text (≥3:1).
- **Performance**: Zero runtime cost — all changes are static CSS / static-rendered React.
- **Theming**: Both light and dark themes pass the same accessibility bar.
- **Maintainability**: Numeric counts have one source of truth (`generated-counts.ts`).
