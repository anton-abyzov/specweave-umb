# Architecture Plan — 0865 Rubric-Live + Sync-Hardening

All paths below are relative to `repositories/anton-abyzov/specweave/`.

## Design principle

A rubric is the **per-increment acceptance contract**: one independently-gradeable, objectively-verifiable criterion per AC, scored pass/fail by a blind evaluator, with blocking failures hard-gating `/sw:done`. Wherever a criterion can be checked deterministically (grep/test/script), it should be a `command` criterion (cheap, reproducible, no LLM). LLM-judge evaluators (`grill`/`code-reviewer`/`judge-llm`) are reserved for intent/semantics. This increment closes the two structural holes that make that impossible today (generator unwired + only-LLM evaluators) and dogfoods the result on the sync surface.

## Part A — Rubric engine goes live

### A1. Wire the generator into the lifecycle (FR-001, AC-US1-01/02)
- `generateRubricFile` (`src/core/rubric/rubric-generator.ts:146`) currently has no caller. Add invocation **after `spec.md` has ACs** — at `tasks.md` write / end-of-planning — writing to the increment **ROOT** `rubric.md` (the path `completion-validator.ts:199` reads), not `reports/rubric.md`.
- Add CLI `specweave generate-rubric <id> [--refresh]` (`src/cli/commands/`): idempotent by default (no clobber if a non-template rubric exists), `--refresh` regenerates from current ACs.
- Generator maps each `AC-USx-yy` → one `### R-USx-yy: <title> [blocking]` block with `Source`/`Evaluator`/`Verify`/`Threshold`/`Result` fields, then appends the 4 inherited defaults (R-D01..R-D04).
- Parity: a test asserts `criteria.length === acCount + 4`.

### A2. Deterministic `command` evaluator (FR-002, AC-US1-03)
- Extend the evaluator dispatch in `rubric-evaluator.ts:125-145` with a `command` case.
- Execution: run the criterion's `Verify` string via `child_process` with `cwd` = repo root (or increment dir), a bounded timeout (default 60s, configurable), and `env` stripped of credential vars (do not inject tokens). Map exit 0 → `pass`, non-zero/timeout → `fail`. Never `skip`. Network is NOT blocked (legitimate probes like `npx vitest` need it); OS-level isolation is a documented follow-up. The committed/reviewed rubric is the trust boundary.
- Capture a truncated stdout/stderr tail as `evidence`.

### A3. Closure gating (FR-003, AC-US1-04)
- `completion-validator.ts` already pushes blocking failures into the error gate after `evaluateRubric`. Add/confirm a unit test that a failing **blocking** `command` criterion blocks closure and a failing **advisory** one does not.

### A4. Legacy removal + scaffolder fix (FR-004, AC-US1-05)
- Delete the 143 existing `.specweave/increments/*/rubric.md` template/unparseable files (umbrella root) wholesale per Anton's decision.
- `template-creator.ts:357-377`: stop writing the `reports/rubric.md` placeholder. The root `rubric.md` is produced by A1 post-planning instead.

## Part B — Sync hardening (proving ground)

### B1. Routing source-of-truth (FR-005, AC-US2-01)
- Rewrite `src/sync/sync-target-resolver.ts`, `story-router.ts`, and `src/cli/commands/sync-progress.ts` to consume `config.workspace.repos[]` (via `external-tool-resolver.ts`) and **delete every `config.umbrella` read** in `src/sync`. After the 0625 migration `config.umbrella` is undefined → today the resolver silently collapses to the global fallback, losing child-repo routing.

### B2. One credential resolver (FR-006, AC-US2-02/03)
- Extract a single `CredentialResolver` imported by **both** `src/cli/commands/sync-health.ts` and the ADO runtime resolver (`src/integrations/ado/ado-pat-provider.ts`), plus Jira/GitHub.
- Canonical alias table — ADO: primary `AZURE_DEVOPS_PAT`, aliases `AZURE_DEVOPS_TOKEN`, `ADO_PAT`. Health and runtime now agree, closing the green-health/failing-write divergence.
- Secrets read via `sync-setup`/Obsidian; resolver never logs values; a credential-masker stays the last write barrier.

### B3. Resilient writes (FR-007, AC-US2-04)
- Wrap `ProviderAdapter` external write calls in a retry-wrapper (bounded backoff). On terminal failure, auto-enqueue into `SyncRetryQueue` so `sync-retry` drains a populated queue. Collapse to one persistent circuit breaker if duplicates exist.

### B4. Deprecated skill (AC-US2-05)
- Remove or reconcile the `github-sync` skill (`plugins/specweave/skills/github-sync/SKILL.md`) against the live increment-based github command so guidance is not contradictory.

### B5. Dogfood rubric (AC-US2-06)
- Ship this increment's own `rubric.md` (root) with `command` criteria for the grep/test invariants (advisory for sync per the rollout decision), `sw:grill`/`sw:code-reviewer` criteria for semantics, generated by the now-live generator. `/sw:done` evaluates them.

## Risks & mitigations

- **Arbitrary shell at closure** → credential-stripped env + bounded timeout; rubric is committed/reviewed (code-review gate), so the reviewed rubric is the trust boundary. OS-level network isolation is a documented follow-up (not enabled — legitimate probes like `npx vitest` need the network).
- **Mis-scoped grep criterion locks repo-wide closures** → sync criteria ship **advisory** first (promote later).
- **`--refresh` clobbers hand-authored criteria** → default generate is non-clobbering; `--refresh` is explicit.
- **Legacy deletion regrets** → files are git-tracked (recoverable); they carried no real gating content.
