# Feature: Rubric-Live + Sync-Hardening

## Overview

SpecWeave shipped a rubric engine in increment 0663 (`src/core/rubric/`), but it is dead weight. The generator that emits the only parseable criterion format has **zero production callers**, the scaffolder writes a placeholder to a file the closure gate never reads, and the evaluator can only run LLM-judge evaluators — every deterministic check returns `skip`. Net effect: **0 of 143 per-increment rubrics are in the gating format**, so `/sw:done` always silently falls back to the same 4 hardcoded default gates. The rubric is a no-op.

This increment makes the rubric **live**: the generator runs during planning and writes one blocking criterion per AC to the path the gate reads; a new **`command` evaluator** runs a criterion's shell probe in a sandbox and hard-blocks closure on failure. The **external-sync flow (GitHub / Jira / ADO)** is the proving ground — it has confirmed reliability defects, and this increment hardens them and ships its own AC-tied, command-evaluated rubric as living proof the engine now gates.

## Background — confirmed gap (deep audit)

- **Read-side is correct**: `completion-validator.ts:199-259` reads the increment **root** `rubric.md`, merges layers (`rubric-merger.ts:43`), evaluates, and pushes blocking failures into the closure error gate.
- **Write-side break #1**: `generateRubricFile` (`src/core/rubric/rubric-generator.ts:146`) — the only code that emits the parser-required `### R-NNN … [blocking]` shape (`rubric-parser.ts:14`) — has **zero production callers**.
- **Write-side break #2**: scaffolding writes a `status: template` placeholder to `reports/rubric.md` (`template-creator.ts:357-377`); the gate reads the **root** `rubric.md` — a different file, skipped as `isTemplate`.
- **Write-side break #3**: `sw-planner` is prose-only and `increment/SKILL.md` instructs a `## Quality Contract` heading the parser reads as zero criteria.
- **Evaluator gap**: `rubric-evaluator.ts:125-145` automates only `sw:grill` / `sw:code-reviewer` / `sw:judge-llm`; `coverage` / `manual` / default all return `skip`. There is no deterministic script/grep/test evaluator, so structural invariants cannot gate.
- **Sync defects (proving ground)**: leftover `config.umbrella` reads in the routing path after the 0625 migration to `config.workspace.repos[]`; an ADO PAT alias gap where `sync-health` accepts `AZURE_DEVOPS_PAT | AZURE_DEVOPS_TOKEN | ADO_PAT` but the runtime resolver accepts only `AZURE_DEVOPS_PAT` (health reports green while real writes throw); a `SyncRetryQueue` that is structurally always empty because no live write path enqueues into it; and a deprecated, contradictory `github-sync` skill.

## Locked Decisions

1. **`command` evaluator = sandbox any command.** Runs the criterion's `Verify` shell probe in the increment cwd with a bounded timeout and a credential-stripped environment (PAT/TOKEN/SECRET/… removed; no tokens injected); `status=pass` on exit 0, `fail` on non-zero, **never `skip`**. The rubric file is a committed, code-reviewed artifact, which is the primary trust boundary. (OS-level network isolation — `unshare -n` / `sandbox-exec` — is a documented follow-up; it is intentionally not enabled by default because legitimate probes such as `npx vitest` may need the network.)
2. **Migration = delete legacy rubrics.** The 143 existing template/unparseable `rubric.md` files are deleted; every new increment must strictly follow the generated `R-NNN` rubric going forward.
3. **Rollout = advisory-then-promote.** The sync-hardening rubric criteria ship **advisory** (reported, non-gating) this increment; the rubric-engine criteria (US-001) are **blocking** and proven by unit tests. Sync criteria promote to blocking in a follow-up after one green cycle.
4. **Opus 4.8 default model.** Set Opus 4.8 as the default model in config; keep `sw-planner` on Sonnet (rubric generation is now deterministic — no Opus needed there).

## Functional Requirements

### FR-001: Rubric generation is wired into the lifecycle
`generateRubricFile` MUST be invoked after `spec.md` has ACs (post-planning / on `tasks.md` write) and write to the increment **root** `rubric.md`. A `specweave generate-rubric <id> [--refresh]` CLI MUST exist for idempotent regeneration.

### FR-002: Deterministic `command` evaluator
The rubric evaluator MUST support a `command` evaluator type that executes the criterion's `Verify` probe in a sandbox (cwd = repo/increment, bounded timeout, credential-stripped env) and maps exit code to `pass`/`fail` with captured evidence (stdout/stderr tail). It MUST never return `skip` for a `command` criterion. (Network isolation is out of scope for this increment — see Locked Decision 1.)

### FR-003: Closure gating
`completion-validator` MUST treat a failed **blocking** criterion (any evaluator, including `command`) as a closure blocker. **Advisory** criteria are reported but never block.

### FR-004: Legacy rubric removal + scaffolder fix
The 143 existing template/unparseable `rubric.md` files MUST be deleted. Scaffolding MUST stop writing a template placeholder to `reports/rubric.md`.

### FR-005: Sync routing source-of-truth
Sync routing MUST read `config.workspace.repos[]` exclusively; no `config.umbrella` reads remain in the routing path.

### FR-006: Single credential resolver
One `CredentialResolver` MUST be the single source of provider credentials, imported by both `sync-health` and the runtime provider adapters, with a canonical alias table (ADO: `AZURE_DEVOPS_PAT` + aliases `AZURE_DEVOPS_TOKEN` / `ADO_PAT`). Values are sourced via `sync-setup` / Obsidian and never hardcoded or logged.

### FR-007: Resilient external writes
External write calls MUST be wrapped so terminal failures auto-enqueue into `SyncRetryQueue`, which `sync-retry` drains. The queue MUST no longer be structurally always-empty.

## Success Criteria

- A newly planned increment has a populated, AC-tied `rubric.md` at its root; `/sw:done` evaluates real criteria, not the 4 defaults.
- A failing `command` probe demonstrably blocks closure (unit-proven).
- `grep -rn "config\.umbrella" src/sync` returns zero matches in the routing path.
- ADO sync-health and runtime resolver share one resolver and accept the same PAT aliases (unit-proven).
- A forced 5xx during a sync write lands a drainable job in `SyncRetryQueue` (unit-proven).
- The 143 legacy rubric files are gone; scaffolding no longer emits a placeholder.
- Zero secret values in any output; zero critical/high code-review findings.
