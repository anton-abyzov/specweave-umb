# Implementation Plan: Studio Parameters & Secrets explainer UX + docs page

## Overview

Pure UI + content change across two repos under the umbrella:

1. **vskill** (eval-ui React bundle) — augment `CredentialManager.tsx` with a header info icon and an enhanced empty state.
2. **vskill-platform** (Next.js docs site at verified-skill.com) — add a static MDX docs page at `/docs/parameters-and-secrets` and let the existing prebuild script auto-register it in the sidebar nav.

No backend changes. No API changes. No new dependencies. Native browser tooltip primitives reused. Existing styling system (CSS variables + Tailwind) kept untouched.

## Architecture

### Components

- **`CredentialManager.tsx`** (`repositories/anton-abyzov/vskill/src/eval-ui/src/pages/workspace/CredentialManager.tsx`): owns both the section header and the empty state. Two surgical edits — header gets a trailing `(i)` `<span>` with `title=""`, empty-state `<div>` is replaced with a 3-block card.
- **MDX docs page** (new at `repositories/anton-abyzov/vskill-platform/src/app/docs/parameters-and-secrets/page.mdx`): static content, follows existing frontmatter conventions.
- **Sidebar nav**: `generated-nav.ts` is regenerated by `scripts/generate-docs-nav.cjs` at prebuild time — no manual edit needed unless the script misses the file.

### Data Model

No data model changes. `CredentialStatus` (types.ts:814) and `IntegrationEvalCase.requiredCredentials` (integration-types.ts:40-50) stay as-is.

### API Contracts

No API changes. Existing endpoints unchanged:
- `GET /api/credentials/:plugin/:skill`
- `PUT /api/credentials/:plugin/:skill`
- `GET /api/credentials/:plugin/:skill/params`

## Technology Stack

- **eval-ui**: React + TypeScript, Vite bundle, native HTML for tooltips/links, CSS variables for theming
- **vskill-platform**: Next.js 15 (App Router), MDX content with prebuild nav generation
- **Tests**: Vitest (eval-ui), `@testing-library/react`

**Architecture Decisions**:

- **Native `title=""` over a Tooltip component**: matches existing pattern at `CredentialManager.tsx:189-190`, zero new deps, zero JS state, accessible by default in screen readers. Trade-off: minimal styling control over the tooltip bubble — acceptable for a low-frequency, informational hint.
- **Empty-state expansion in place vs. modal/drawer**: inline empty-state copy is seen by 100% of new users (no click required), whereas a modal needs an opt-in click. Education at the moment of confusion beats an optional dialog.
- **MDX page over inline help drawer**: a dedicated docs URL is shareable, indexable by LLMs, and reusable from other Studio surfaces in the future.

## Testing Strategy

- **Unit (Vitest)**: render `CredentialManager` with stubbed `getCredentials` for both empty and populated cases; assert DOM structure and link attributes.
- **Build gate (Next.js)**: `npm run build` must succeed — catches MDX syntax + frontmatter errors.
- **Manual visual gate**: Studio preview at `localhost:3157` for the actual user flow.

## Technical Challenges

### Challenge 1: MDX `order` collision
**Solution**: enumerate all sibling page files, pick a non-colliding integer (target ~35, between studio and submitting).
**Risk**: nav out of order. **Mitigation**: visual check after build.

### Challenge 2: prebuild nav script doesn't pick up the new MDX
**Solution**: manual entry in `docs-nav.ts` as fallback.
**Risk**: page exists but is not discoverable. **Mitigation**: verify sidebar contains the entry post-build.

### Challenge 3: Studio bundle is shipped pre-built; users running an installed `vskill` won't see changes until next release
**Solution**: ship via the standard vskill release pipeline (Vite build → npm publish). Acceptable — this is the same constraint every other Studio change has.
